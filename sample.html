§ 1 Overview

Purpose

The Acme Platform provides a unified layer for authentication, data access, and event routing across all product modules. It is designed to be consumed by both internal services and third-party integrations.

Target Audience

  • Backend engineers integrating with platform APIs
  • DevOps engineers managing deployments and infrastructure
  • Product managers reviewing technical constraints

§ 2 Architecture

Components

The platform consists of three primary services:

  • Gateway — entry point for all external requests, handles TLS termination and routing
  • Auth Service — issues and validates JWT tokens, manages OAuth flows
  • Data API — provides read/write access to the canonical data store

Communication

Internal service-to-service calls use gRPC over mTLS. External clients communicate via REST over HTTPS. Asynchronous workflows are handled through an event bus (Kafka-compatible).

§ 3 Scope

This version (v0.4) covers the MVP feature set. Features planned for GA (v1.0) are tracked separately and will be incorporated in a subsequent revision of this document.

IN SCOPE (v0.4)

  • Token issuance and refresh
  • CRUD endpoints for core resources
  • Webhook delivery with retry

OUT OF SCOPE (v0.4)

  • Multi-region failover
  • Real-time streaming API
  • Admin dashboard UI
